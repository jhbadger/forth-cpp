+
  ( a b -- n )
  Adds a and b.

-
  ( a b -- n )
  Subtracts b from a.

*
  ( a b -- n )
  Multiplies a and b.

/
  ( a b -- n )
  Divides a by b. Integer division, truncates toward zero.

mod
  ( a b -- n )
  Remainder of a divided by b.

=
  ( a b -- flag )
  Returns -1 if a equals b, 0 otherwise.

<
  ( a b -- flag )
  Returns -1 if a is less than b, 0 otherwise.

>
  ( a b -- flag )
  Returns -1 if a is greater than b, 0 otherwise.

0=
  ( n -- flag )
  Returns -1 if n is zero, 0 otherwise.

and
  ( a b -- n )
  Bitwise AND of a and b.

or
  ( a b -- n )
  Bitwise OR of a and b.

xor
  ( a b -- n )
  Bitwise XOR of a and b.

invert
  ( n -- ~n )
  Bitwise NOT of n. Inverts all bits.

lshift
  ( n amt -- n )
  Shifts n left by amt bits.

rshift
  ( n amt -- n )
  Shifts n right by amt bits (logical, zero-fills from left).

>r
  ( n -- )
  Moves top of data stack to return stack.

r>
  ( -- n )
  Moves top of return stack to data stack.

r@
  ( -- n )
  Copies the top of the return stack to the data stack without removing it.

dup
  ( n -- n n )
  Duplicates the top of stack.

drop
  ( n -- )
  Discards the top of stack.

swap
  ( a b -- b a )
  Swaps the top two stack items.

over
  ( a b -- a b a )
  Copies the second item to the top.

rot
  ( a b c -- b c a )
  Rotates the third item to the top.

depth
  ( -- n )
  Pushes the number of items currently on the data stack.

pick
  ( ... n -- ... val )
  Copies the nth item from the top of stack (0-indexed).
  0 pick is equivalent to dup.

.
  ( n -- )
  Prints top of stack in the current base, followed by a space.

emit
  ( c -- )
  Prints a character by its ASCII value.

cr
  ( -- )
  Prints a newline.

.s
  ( -- )
  Prints the entire stack contents without consuming them.

type
  ( idx -- )
  Prints the string at the given string table index.

s=
  ( idx1 idx2 -- flag )
  Returns -1 if the two strings are equal, 0 otherwise.

s+
  ( idx1 idx2 -- idx3 )
  Concatenates two strings, returns index of the new string.

s.s
  ( -- )
  Prints the entire string table with indices. Useful for debugging.

s.clear
  ( -- )
  Clears the string table entirely.

accept
  ( maxlen -- idx )
  Reads a line of input, truncated to maxlen characters.
  Returns the string table index of the input.

key
  ( -- c )
  Reads a single keypress without echo. Returns ASCII value.

base
  ( -- addr )
  Pushes the heap address of the BASE variable.

hex
  ( -- )
  Sets the numeric base to 16.

decimal
  ( -- )
  Sets the numeric base to 10.

octal
  ( -- )
  Sets the numeric base to 8.

binary
  ( -- )
  Sets the numeric base to 2.

@
  ( addr -- n )
  Fetches the integer value at byte address addr.

!
  ( n addr -- )
  Stores integer n at byte address addr.

c@
  ( addr -- n )
  Fetches a single byte from byte address addr.

c!
  ( n addr -- )
  Stores the low byte of n at byte address addr.

cell
  ( -- n )
  Pushes the size of one cell in bytes (4 on this platform).

cell+
  ( addr -- addr+cell )
  Adds one cell size (4 bytes) to an address.

cells
  ( n -- n*cell )
  Converts a cell count to a byte count. Multiplies by cell size.

,
  ( n -- )
  Appends one cell (4 bytes) to the heap at the current HERE position.

c,
  ( n -- )
  Appends a single byte to the heap at HERE.

here
  ( -- addr )
  Pushes the byte address of the next free heap location.

allot
  ( n -- )
  Reserves n bytes on the heap, advancing HERE by n.

fill
  ( addr len val -- )
  Fills len bytes starting at addr with the low byte of val.

dump
  ( addr len -- )
  Prints len bytes starting at addr.

i
  ( -- n )
  Pushes the current loop index. Only valid inside a DO loop.

j
  ( -- n )
  Pushes the outer loop index. Only valid inside a nested DO loop.

unloop
  ( -- )
  Discards the current loop's index and limit from the return stack.
  Required before EXIT inside a DO loop.

compile,
  ( xt -- )
  Compiles a call to the word identified by execution token xt.

postpone
  ( "name" -- )
  Compiles the compilation semantics of name into the current definition.
  For immediate words: compiles a call that runs at the enclosing word's
  compile time. For non-immediate words: compiles code that will compile
  a call to name when the current word runs.

execute
  ( xt -- )
  Executes the word identified by execution token xt.

immediate
  ( -- )
  Marks the most recently defined word as immediate.
  Immediate words execute at compile time rather than being compiled.

words
  ( -- )
  Prints all known words in the dictionary.

'
  ( -- xt )
  Syntax: ' wordname
  Pushes the execution token of the named word.
  In compile mode, compiles the xt as a literal.

see
  ( -- )
  Syntax: see wordname
  Decompiles and prints the bytecode of the named word.

include
  ( -- )
  Syntax: include filename
  Loads and evaluates a Forth source file.

edit
  ( -- )
  Syntax: edit filename
  Opens a file in the system editor (uses $EDITOR, falls back to vi).

bye
  ( -- )
  Exits the Forth interpreter.

:
  ( -- )
  Syntax: : name ... ;
  Begins a word definition. Everything until ; is compiled.

;
  ( -- )
  Ends a word definition and stores it in the dictionary.

create
  ( -- )
  Syntax: create name
  Creates a new named dictionary entry whose body is the current HERE.
  When invoked, pushes its body address. Typically used with DOES>.

does>
  ( -- )
  Splits a defining word into build and run phases.
  Code before DOES> runs when the defining word is invoked.
  Code after DOES> runs whenever the created word is invoked.

if
  ( flag -- )
  Begins a conditional. If flag is 0, skips to ELSE or THEN.
  Only valid inside a definition.

else
  ( -- )
  Separates the true and false branches of an IF...THEN block.
  Only valid inside a definition.

then
  ( -- )
  Closes an IF or IF...ELSE block.
  Only valid inside a definition.

endif
  ( -- )
  Alias for THEN. Only valid inside a definition.

do
  ( limit start -- )
  Begins a counted loop from start to limit-1.
  Use I for the current index. Only valid inside a definition.

loop
  ( -- )
  Increments the loop index by 1 and jumps back to DO if index < limit.
  Only valid inside a definition.

+loop
  ( n -- )
  Increments the loop index by n and jumps back to DO if not past limit.
  Only valid inside a definition.

leave
  ( -- )
  Immediately exits the enclosing DO loop.
  Only valid inside a definition.

begin
  ( -- )
  Marks the start of a BEGIN loop.
  Only valid inside a definition.

until
  ( flag -- )
  Jumps back to BEGIN if flag is 0. Exits loop if flag is non-zero.
  Only valid inside a definition.

while
  ( flag -- )
  If flag is 0, exits a BEGIN...WHILE...REPEAT loop.
  Only valid inside a definition.

repeat
  ( -- )
  Unconditionally jumps back to BEGIN in a BEGIN...WHILE...REPEAT loop.
  Only valid inside a definition.

again
  ( -- )
  Unconditionally jumps back to BEGIN. Creates an infinite loop.
  Only valid inside a definition.

recurse
  ( -- )
  Compiles a call to the word currently being defined.
  Only valid inside a definition.

exit
  ( -- )
  Returns immediately from the current word.
  If inside a DO loop, use UNLOOP first.
  Only valid inside a definition.

{
  ( ... -- )
  Syntax: { a b c -- }
  Declares local variables for the current definition.
  Values are popped from the stack in reverse order.
  Only valid inside a definition.

->
  ( n -- )
  Syntax: -> localname
  Stores the top of stack into the named local variable.
  Only valid inside a definition containing locals.

char

  Pushes the ASCII value of the first character of the next token.

[char]

  Compiles the ASCII value of the first character of the next token.
  Only valid inside a definition.

."

  Syntax: ." some text"

.(
  ( -- )
  Syntax: .( some text)

\
  ( -- )
  Line comment. Ignores everything to the end of the line.

(
  ( -- )
  Block comment. Ignores everything until the closing ).

help

  Syntax: help <word>
  Displays help for a word. Bare HELP lists all documented words.

  Stores a help entry for a word. Both arguments are string table indices.

at-xy
  ( col row -- )
  Moves the terminal cursor to the given column and row.
  Origin is (0,0) at the top-left of the terminal.
  Uses ANSI escape sequences; requires an ANSI-compatible terminal.
  Example: 10 5 at-xy  moves cursor to column 10, row 5

state
  ( -- addr )
  Pushes the address of the STATE variable in the heap.
  STATE holds 1 if the interpreter is in compile mode, 0 if in
  interpret mode. Use @ to fetch the current value.
  STATE is set to 1 when : begins a definition and back to 0
  when ; ends it. It is also reset to 0 if an error occurs.

info
  ( -- )
  BadgerForth -- a lightweight Forth interpreter written in C++.
  Architecture:
    Compiles to an internal bytecode (Op enum) dispatched via switch.
    Each instruction has an opcode, an integer field, and a string field.
  Memory model:
    A byte vector (heap) serves as the address space.
    Cell size is 4 bytes (sizeof int). BASE is at heap[0], STATE at heap[4].
    HERE and ALLOT operate on byte addresses.
  String model:
    Strings are stored in a separate string_table vector.
    Most string words operate on integer indices into this table.
    s" pushes an index, TYPE prints by index, s+ concatenates.
  Dictionary:
    Words are stored in a deque<Entry> for pointer stability.
    A name->index map provides O(1) lookup.
    Four kinds of entry: PRIM, WORD, DEFINING, and CREATE.
    Each entry has a unique integer execution token (xt).
  Locals:
    Supported via { a b c -- } syntax inside definitions.
    Locals are stored in a separate lframes stack.
  Compilation:
    Words compile to Op::Call (name-based) during compilation.
    At ; time a resolve pass patches Call -> CallDirect (index-based)
    for all words already in the dictionary, giving O(1) dispatch.
    Unresolved calls (forward references) remain as Op::Call and
    resolve via dictionary lookup at runtime.
  Control flow:
    IF ELSE THEN, BEGIN UNTIL, BEGIN WHILE REPEAT, BEGIN AGAIN,
    DO LOOP, DO +LOOP, LEAVE, EXIT, RECURSE.
  REPL:
    Lines containing only expressions are wrapped in an anonymous
    definition and executed, allowing IF, DO, loops etc. at the prompt.
    Lines starting a definition (:), containing CREATE, or invoking a
    DEFINING word are passed through unwrapped.
